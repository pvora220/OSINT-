# Security Updates - March 14, 2026

## Phase 1 & 2 Implementation Complete

Your MATRIX OSINT application has been upgraded with critical security improvements:

### ✅ What Changed

#### 1. **Admin Credentials Removed from UI**
- **Before**: Default credentials `potato2002 / Potato/2002` were hardcoded and visible in the login screen
- **After**: Credentials no longer displayed in the UI

#### 2. **Password Hashing Implemented**
- **Before**: Passwords stored in plaintext in the database (critical vulnerability)
- **After**: Passwords hashed using PBKDF2 (100,000 iterations) with cryptographic salts
- Uses Node.js built-in `crypto` module (no external dependencies)

#### 3. **Server-Side Session Tokens**
- **Before**: Client sent username/role directly with each request (allows privilege escalation)
- **After**: Secure session tokens generated by server and validated server-side
- Admin role cannot be spoofed by client anymore
- Sessions expire after 24 hours

#### 4. **Admin Account Bootstrap Secured**
- **Before**: Hardcoded `potato2002` was force-reset on every server start
- **After**: Admin account created only once on first run, never overwritten

### 🔑 How to Access Admin Account

**On First Run:**
The server will automatically create an admin account. The password will be printed to the console:

```
Admin account created. Password: <random-hex-string>
```

**Copy this password and save it somewhere secure.**

**For Custom Admin Password:**
Set the `ADMIN_PASSWORD` environment variable before starting the server:

```powershell
# Windows PowerShell
$env:ADMIN_PASSWORD = "your-secure-password"
node server.js
```

**Or in a batch file (start-matrix-osint.bat):**
```batch
@echo off
set ADMIN_PASSWORD=your-secure-password
node server.js
pause
```

### 🔐 Security Improvements Summary

| Issue | Before | After |
|-------|--------|-------|
| Default Creds | Visible in UI | Not displayed |
| Password Storage | Plaintext | PBKDF2 Hashed + Salted |
| Authorization | Client-trusted role | Server-validated sessions |
| Admin Hardcoding | Force-reset every start | One-time bootstrap only |
| Session Hijacking | Easy (no validation) | Impossible (server tokens) |

### ⚠️ Breaking Changes for Developers

If you have custom scripts or integrations:

**Login response now includes `token`:**
```json
{
  "token": "abc123...",
  "user": {"username": "admin", "role": "admin"},
  "loginAt": "2026-03-14T..."
}
```

**Activity logging now requires `token` instead of `username/role`:**
```javascript
// Old (broken):
await fetch('/api/activity', {
  body: JSON.stringify({
    username: "admin",
    role: "admin",
    action: "TEST"
  })
});

// New (correct):
await fetch('/api/activity', {
  body: JSON.stringify({
    token: sessionToken,
    action: "TEST"
  })
});
```

**Logout endpoint added:**
```javascript
await fetch('/api/auth/logout', {
  method: 'POST',
  body: JSON.stringify({ token: sessionToken })
});
```

### 📞 Troubleshooting

**"Invalid credentials" when logging in?**
- Server has restarted with new database. Use the password printed in console on first run.
- OR: Delete `data/matrix-osint.db` to reset everything and get a new admin password.

**Can't find the admin password?**
- Check the terminal window where you ran `node server.js` - it scrolls past quickly
- Delete the database and restart to get a new password
- Set `ADMIN_PASSWORD` environment variable before starting

**Old user accounts not working?**
- Existing users with plaintext passwords won't authenticate
- They can register new accounts or reset their password through registration

**Deployment Notes**: For production use, implement HTTPS, use Redis for session storage (not in-memory), and integrate with a secrets manager for the admin password.
